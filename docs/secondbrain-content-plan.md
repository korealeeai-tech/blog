# SecondBrain 콘텐츠 계획

이 문서는 SecondBrain 카테고리에 올릴 글감을 공개 가능한 수준으로 정리한 계획이다.

SecondBrain은 여기서 특정 내부 시스템의 원문 로그나 구현 세부를 공개하는 이름이 아니다. AI가 사용자의 의도를 더 안정적으로 이해하도록 돕는 개인화된 판단 기준 라이브러리, 기억 구조, 검증 루프를 설명하기 위한 공개용 개념으로 다룬다.

## 공개 원칙

- 실제 대화 원문, 개인 정보, 내부 경로, 비공개 작업 맥락은 공개하지 않는다.
- 사례는 합성 예시로 바꿔 쓴다.
- Graph나 dashboard 이미지는 기본적으로 공개용 다이어그램으로 만든다. 사용자가 명시 승인한 경우 실제 화면 캡처를 사용할 수 있지만, 내부 지식 이름, 원문 경로, 작업명, 계정, 로그, queue/schedule 세부값은 블러 또는 마스킹한다.
- SecondBrain을 사용자를 분석하거나 대체하는 시스템으로 설명하지 않는다.
- 핵심 메시지는 "더 많이 기억하는 AI"가 아니라 "더 올바른 기준으로 판단하는 AI"에 둔다.

## 올릴 만한 주제

### 1. 사용자 의도 해석 문제

AI가 같은 사용자와 계속 일해도 매번 요청을 처음 보는 것처럼 해석하는 문제를 다룬다. 프롬프트 한 문장만으로는 사용자의 선호, 검증 기준, 반복 실수 회피 기준을 충분히 알기 어렵다는 점을 설명한다.

### 2. SecondBrain을 의도 라이브러리로 보기

SecondBrain을 단순 메모장, 대화 기록 저장소, 사용자를 감시하는 구조가 아니라 AI가 작업 전에 참고하는 판단 기준 라이브러리로 설명한다. 사용자는 SecondBrain을 "나를 대신하는 AI"가 아니라 "AI가 내 의도를 덜 오해하도록 돕는 기준층"으로 이해한다.

### 3. SecondBrain Graph

기억을 날짜순 로그로 쌓는 대신 선호, 검증 규칙, 위험, 작업 맥락, 응답 전략을 노드와 관계로 연결하는 구조를 설명한다. 실제 graph 캡처는 민감 영역을 블러 처리하고, 개념 설명은 합성 노드와 관계를 사용한다.

### 4. Pre-work Memory Brief

작업 시작 전에 관련 기억 후보, 확인 필요한 가정, 검증 경로를 먼저 꺼내는 흐름을 설명한다. 핵심은 기억을 확정 사실로 쓰는 것이 아니라 source check가 필요한 항목을 분리하는 것이다.

### 5. Work Impact Tracking

SecondBrain이 실제로 도움이 됐는지 관찰하는 방법을 다룬다. 검증 강화, 위험 감소, 반복 설명 감소, 추적성 개선, 작업 전 맥락 회수 같은 축으로 설명한다.

### 6. 자동 성장 루프

새 작업에서 얻은 피드백과 사고 방지 기준을 후보로 모으고, 검토 후 반영하는 루프를 설명한다. 자동으로 다 믿는 것이 아니라 후보, 검토, 승격, 보류를 나누는 구조가 핵심이다.

### 7. Promotion Review Queue

좋은 기억과 나쁜 기억을 가르는 gate를 다룬다. 오래된 추측, 과잉 일반화, 민감정보, 사용자를 고정적으로 단정하는 기억은 승격하지 않아야 한다.

### 8. Turn-level Knowledge Capture

대화 한 턴에서 새롭게 발견한 사용자 선호나 작업 기준을 후보로 잡는 구조를 설명한다. 단, 후보 수집과 공식 기억 반영은 분리해야 한다.

### 9. 사고 방지 기억

검증하지 않은 완료 보고, 경로 추측, stale 상태 판단 같은 반복 오류를 줄이는 기억 구조를 설명한다. 독자 공감도가 높고 실용성이 좋다.

### 10. Graph Visualizer와 업무효과 Dashboard

SecondBrain이 어떤 기준을 갖고 있고 어떤 종류의 도움을 줬는지 시각화하는 방법을 설명한다. 실제 데이터 대신 샘플 데이터를 써서 공개 안전성을 유지한다.

### 11. 외부 공개 안전장치

개인화 memory를 블로그나 보고서에 공개할 때 지켜야 할 경계를 설명한다. 원문 대화, 내부 경로, raw log, queue, graph 세부값은 공개하지 않는다는 원칙을 다룬다.

### 12. 좋은 기억과 나쁜 기억

좋은 기억은 근거가 있고, 갱신 가능하며, 검증 기준과 연결된다. 나쁜 기억은 오래된 추측, 과잉 일반화, 민감정보, 사용자를 고정적으로 단정하는 기억이다.

## 추천 우선순위

### 1편. AI는 왜 사용자의 의도를 매번 새로 추측할까

문제 제기 글이다. AI memory의 필요성을 기술 설명보다 경험적 문제에서 출발해 설명한다.

핵심 메시지:

- 프롬프트만으로는 사용자의 반복 기준을 알기 어렵다.
- 같은 요청도 사용자마다 원하는 답이 다르다.
- AI가 기억해야 할 것은 원문 로그가 아니라 판단 기준이다.

### 2편. SecondBrain: AI를 위한 사용자 의도 라이브러리

SecondBrain을 어떻게 이해하고 있는지 명확히 설명하는 글이다.

핵심 메시지:

- SecondBrain은 사용자를 복제하는 시스템이 아니다.
- SecondBrain은 사용자의 판단 기준을 AI가 참고할 수 있게 정리한 라이브러리다.
- 기억은 결론이 아니라 검증을 시작하는 단서다.

### 3편. SecondBrain Graph: 기억을 연결해야 의도가 보인다

Graph 시각화를 처음 본격적으로 보여주는 글이다.

핵심 메시지:

- 날짜순 로그보다 관계형 기억 구조가 의도 해석에 더 유용하다.
- 선호, 검증 규칙, 위험, 맥락, 응답 전략은 서로 연결되어야 한다.
- 실제 Graph 캡처는 민감 영역을 블러 처리하고, 구조 설명은 합성 그래프로 보완한다.

상태: 게시 완료. `src/content/blog/secondbrain-graph-memory-network.md`

### 4편. 작업 전에 기억을 꺼내는 방법: Pre-work Memory Brief

SecondBrain이 작업 시작 전에 어떻게 작동해야 하는지 설명한다.

핵심 메시지:

- 관련 기억 후보와 확인 필요한 가정을 분리한다.
- 기억을 확정 사실로 쓰지 않고 source check와 연결한다.
- 사용자가 반복해서 설명하던 맥락을 줄일 수 있다.

### 5편. AI memory가 실제로 도움이 됐는지 측정하는 법

SecondBrain의 효과를 감으로 말하지 않고 관찰 축으로 나누는 글이다.

핵심 메시지:

- 좋은 memory는 검증, 위험 감소, 반복 설명 감소에 영향을 준다.
- 업무효과 dashboard는 source proof가 아니라 관찰 요약이다.
- 효과 주장은 실제 검증 증거와 분리해 다뤄야 한다.
